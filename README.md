# 初級IT研修を“仕組み”で支える ―― 単一HTML学習ハブ「うさうさ フィエスタ学習工房」

> 対象：新人IT研修の受講者／サポートする講師
> レベル目安：🟢 はじめての方も／🟡 少し慣れてきた方／🔴 つまずきがちな所
> 一言で：**ブラウザで開くだけ**で、作図・メモ・勉強会準備・学習までを一つにまとめた学習ハブです。

---

## 概要

研修の現場では「資料はどこ？」「図はどう描くの？」「これ何を勉強すれば…？」が日々くり返されます。
このツールは、それらを**1つのHTMLファイル**に集約し、つまずきを“個人の頑張り”ではなく**仕組み**で減らすことを狙ったものです。

「人ではなく仕組み」――迷う場所を先に用意しておけば、受講者も講師もラクになります。

---

## WHY ―― なぜ作ったか

研修支援でよくある困りごとを、3つに分けて整理しました（モレなく・ダブりなく）。

- **散らばる**：図ツール、メモ、テンプレ、調べもの先がバラバラ。
- **詰まる**：「何から始める？」「エラーの意味は？」で手が止まる。
- **続かない**：スキマ時間に学べる入口がない。

→ ならば、**入口を1か所に**まとめてしまおう、というのが出発点です。

---

## WHAT ―― 何ができるか

タブで機能が分かれています。代表的なものを難易度つきで紹介します。

| タブ | できること | 目安 |
|---|---|---|
| 🔀 作図 | フロー／ER／クラス／シーケンス／状態／**データ設計**を、書きながらプレビュー | 🟡 |
| 📝 メモ | Markdownで書いて、見ながら整える | 🟢 |
| 📤 出力センター | メモから **PowerPoint／Word／PDF／Markdown／CSV** を下書き生成 | 🟡 |
| 🐰 AIうさうさ | 学習サポート（3つの口調）。回答の**音声読み上げ**も | 🟢 |
| 🧩 MECEサポート | 目的を選ぶと「最適な図＋やること3ステップ」 | 🟡 |
| 🗄️ DB設計 | 列を足すと **定義書／CREATE TABLE／ER図** を生成 | 🔴 |
| 🧰 ツールボックス | chmod計算機・正規表現・JSON整形・Base64・cron解説 | 🟡 |
| 🛡️ 設計書レビュー | セキュリティ観点の自動チェック＋チェックリスト | 🔴 |
| 🧭 迷ったらマップ | 「いまの困りごと」から3ステップを提示 | 🟢 |
| 🎲 暇つぶし学習 | 一問一答・ミニクイズ・**学習ダッシュボード** | 🟢 |

ほかにも、掲示板・参考URL・困ったら（セキュリティのチートシート含む）・使い方タブがあります。

---

## HOW ―― どう使うか（研修フロー例）

研修の流れにそって、無理なく差し込めます。

1. **導入**：🧭 迷ったらマップで「今日やること」を1つに絞る。
2. **理解**：🔀 作図で、手順や構造を“描いて”確かめる。
3. **整理**：📝 メモに WHY→WHAT→HOW で書き、📤 出力センターで配布物に。
4. **定着**：🎲 暇つぶし学習で1問。ダッシュボードで継続を見える化。

### 文系トラック／理系トラックの両対応

- **文系・はじめての方**：まず比喩や「迷ったらマップ」から。図は近いテンプレを読み込んで真似る。
- **技術寄りの方**：DB設計・ツールボックス・設計書レビューで手を動かす。

---

## 安心して使える設計（セキュリティ）

研修の成果物に**実名や機密が混ざらない**ことは外せません。そこも仕組みで守ります。

- 🔐 **機密・実名スキャナ**：保存・出力・AI送信・掲示の前に、APIキーやパスワード、氏名らしき表記を検知して警告。役職名（メイン講師・サブ講師・人事担当 など）はOK。
- 🔒 **CSP（外部送信の制限）**：データの送信先を必要な範囲だけに制限し、持ち出しを構造的に防止。
- 🧼 **安全な表示**：画面に出す文字列は必ずエスケープ（XSS対策）。

## たとえ話（※ 正確ではありません）

このツールは「🍜 うさうさラーメン店」に似ています。
券売機（メモ）で注文を決め、厨房（作図・出力）で形にし、出前（PDF/PPT）で届ける――という流れです。
ただし**この比喩は不正確**です。実際にはデータは店内（この端末）だけで完結し、外へ運ぶのは指示したときだけ、という違いがあります。比喩は入口に留め、正確な動きは本文で押さえてください。

## 事実と所感（分けて書きます）

### 事実
- 単一HTMLで動作し、AI以外はオフラインで完結する版（同梱）と、軽量なアーティファクト版（図ライブラリのみCDN）の2系統がある。
- 機密・実名スキャン、CSP、入力エスケープを実装している。
- 公開前に、単体・結合・総合・ホワイトボックス・ブラックボックス・モンキーの観点で**168項目の自動テストに合格**（0件の不具合）している。
- PowerPoint出力は、広く使われるライブラリで生成し、実際にファイルを開いて妥当性を確認している。

### 所感
- 図やAI、音声は便利だが、最終的な体裁チェックや内容の正しさは**人の目**が要る、という設計思想にしている。
- 機械判定（実名・機密・セキュリティ観点）はあくまで“ヒント”。見落としも誤検知もあり得るので、過信は禁物だと感じる。
- 「入口を1か所に」しただけで、迷いが減る手応えがあった。続きは現場の声で磨きたい。
